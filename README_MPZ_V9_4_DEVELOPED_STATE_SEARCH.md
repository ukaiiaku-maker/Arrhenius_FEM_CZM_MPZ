# MPZ v9.4 common developed-state search

## Purpose

The v9.4 Peierls–Taylor screen identified one strict transport closure that is admissible for ceramic-intrinsic, weak-temperature, and DBTT-precursor emission surfaces. The next stage holds that PT closure and all intrinsic EXP-floor barriers fixed while searching one common moving-process-zone state closure.

This is a reduced 1-D moving-front search, not a virgin first-passage fit and not a 2-D FEM/CZM production run. It tests whether the selected intrinsic hazards can evolve toward the intended brittle, moderate-R-curve, and DBTT branches through explicit source occupation, transport, retention, recovery, shielding, blunting, and moving-frame renewal.

## Parameters searched

Only active production MPZ parameters are varied:

- source sites per system;
- trapping barrier;
- retained-population recovery barrier;
- blunting coefficient;
- source recovery rate;
- source refresh length;
- moving-process-zone length.

The fixed legacy `mpz_glide_barrier_eV` and `mpz_detrap_barrier_eV` fields are not searched. Production glide and retained release are controlled by the selected emission-derived detailed-balance Peierls–Taylor kinetics.

The same sampled MPZ closure is applied to all three intrinsic rows. This prevents class separation from being generated by unrelated class-specific bulk/process-zone laws.

## Default screen

The default production screen evaluates:

```text
ceramic: 300, 900, 1200 K
weakT:   300, 700, 1200 K
DBTT:    300, 700, 900, 1200 K
```

Each reduced R-curve advances to 400 micrometres in 5 micrometre increments. The score uses only developed R-curve and state constraints; it does not require a developed DBTT state at virgin first passage.

For each run the code records plateau resistance, R-curve increment, shielding, blunting, retained/mobile populations, source depletion, and approximate convergence lengths for K, shielding, blunting, and retained content.

## Smoke run

```bash
OUTROOT=runs/mpz_v9_4_developed_state_smoke \
SAMPLES=4 \
MAX_WORKERS=1 \
TARGET_EXTENSION_UM=100 \
TEMPERATURES="ceramic:300,1200;weakT:300,1200;DBTT:300,900,1200" \
bash run_mpz_v9_4_developed_state_search.sh
```

## Production run

```bash
nohup env \
OUTROOT=runs/mpz_v9_4_developed_state_search_v1 \
SAMPLES=128 \
MAX_WORKERS=2 \
TARGET_EXTENSION_UM=400 \
DA_UM=5 \
DK=0.25 \
KDOT=0.005 \
KMAX=65 \
bash run_mpz_v9_4_developed_state_search.sh \
> runs/mpz_v9_4_developed_state_search_v1.nohup.log 2>&1 &
```

The search writes one checkpoint per common closure and resumes automatically.

## Outputs

- `developed_state_search_all.csv`: aggregate score for every common closure;
- `developed_state_search_metrics.csv`: class/temperature metrics for every evaluated closure;
- `developed_state_search_shortlist.csv`: ranked common closures;
- `developed_state_shortlist_metrics.csv`: detailed metrics for top closures;
- `developed_state_shortlist_events.csv`: R-curve and state trajectories for top closures;
- `developed_state_candidate_*.png`: K, shielding, blunting, and retained-state evolution;
- `developed_state_search_config.json`: exact search definition;
- `checkpoints/`: resumable per-candidate results.

Passing this stage means that a common reduced MPZ closure produces the intended developed-state topology and converges over a finite extension. It still requires convergence refinement and low/transition/high-temperature FEM/CZM validation.
