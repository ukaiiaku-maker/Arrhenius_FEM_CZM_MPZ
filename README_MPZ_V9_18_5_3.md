# MPZ v9.18.5.3 — quality-selected Mode-I corridor

## Failure resolved

The v9.18.5.2 startup audit correctly rejected the fixed 25 um multi-center
corridor at the 60 um ceramic gate:

- measured minimum triangle quality: `0.03035725`
- required production floor: `0.035`

The failure occurred before FEM assembly and before any physical event.  The
quality gate therefore behaved correctly; lowering the floor would only hide a
mesh-construction defect.

## v9.18.5.3 method

The initial corridor is now selected from a deterministic set of candidate
center counts.  For every candidate the code:

1. builds the multi-center radial Delaunay mesh;
2. removes unused Delaunay input nodes and remaps connectivity;
3. verifies positive finite element areas and no orphan nodes;
4. measures the minimum global triangle quality;
5. samples the local `hbar_tip` along the entire committed target plus guard at
   physical crack-increment spacing;
6. accepts only candidates satisfying both
   `q_min >= ARRHENIUS_MIN_INITIAL_TRIANGLE_QUALITY` and
   `max(hbar_tip/da) <= ARRHENIUS_MAX_TIP_H_OVER_DA`;
7. selects the candidate with the largest quality margin, then the largest
   resolution margin, then the lower node count.

For the 60 um target plus 10 um guard, the candidate set includes a three-center
corridor at 0, 35, and 70 um relative to the initial notch.  This avoids the
near-coincident ring points generated by the prior 23.333 um effective spacing
while retaining the required resolution between centers.
